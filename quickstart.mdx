Get Taskcore running locally in under 5 minutes using Docker Compose. This guide will help you start the application, create your first user, and access the web interface.

## Prerequisites

You need the following installed on your machine:

- **Docker** - version 20.10 or later
- **Docker Compose** - version 2.0 or later
- **Git** - to clone the repository

<Note>
  Taskcore runs a PostgreSQL database and Go backend with embedded SvelteKit frontend. Docker Compose handles all services automatically.
</Note>

## Quick Start

<Steps>
  <Step title="Clone the repository">
    Clone the Taskcore repository to your local machine:

    ```bash
    git clone https://github.com/start-codex/taskcore
    cd taskcore
    ```
  </Step>

  <Step title="Start the application">
    Use Docker Compose to build and start both the database and application:

    ```bash
    docker compose up --build
    ```

    This command will:
    - Pull the PostgreSQL 18 Alpine image
    - Build the Taskcore application image (frontend + backend)
    - Start the database with health checks
    - Start the application server on port 8080
    - Automatically run database migrations

    <Tip>
      The first build may take a few minutes as it installs dependencies and builds both the frontend and backend. Subsequent starts will be much faster.
    </Tip>

    Wait until you see the log message:
    ```
    server listening port=8080
    ```
  </Step>

  <Step title="Create your first user">
    With the application running, create your first user account using the API:

    ```bash
    curl -X POST http://localhost:8080/api/users \
      -H "Content-Type: application/json" \
      -d '{"email": "you@example.com", "name": "Your Name", "password": "yourpassword"}'
    ```

    You should receive a response like:
    ```json
    {
      "status": 200,
      "data": {
        "id": 1,
        "email": "you@example.com",
        "name": "Your Name",
        "created_at": "2026-03-05T10:30:00Z"
      }
    }
    ```
  </Step>

  <Step title="Access the application">
    Open your browser and navigate to:

    ```
    http://localhost:8080
    ```

    Log in with the email and password you just created.
  </Step>
</Steps>

## Stopping the Application

To stop Taskcore and all services:

```bash
# Stop containers but keep data
docker compose down

# Stop containers and remove volumes (deletes all data)
docker compose down -v
```

## Troubleshooting

### Port 8080 already in use

If port 8080 is already taken, you can change it by setting the `APP_PORT` environment variable:

```bash
APP_PORT=3000 docker compose up --build
```

### Database connection issues

If the application fails to connect to the database, check that the database container is healthy:

```bash
docker compose ps
```

You should see both `taskcore-db` and `taskcore-app` with status "running" or "healthy".

### View application logs

To see detailed logs:

```bash
# All services
docker compose logs -f

# Just the application
docker compose logs -f app

# Just the database
docker compose logs -f db
```
